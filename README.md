# 班級黑板 · 三年級數位聯絡簿

專為國小班級經營設計的數位聯絡簿：**直排注音顯示（貼近國語課本）、一鍵投影抄寫、家長手機檢視與數位作業繳交、學藝股長稽核登記、未補繳追蹤**。前端為單一靜態網頁（部署於 Netlify），資料儲存於班級 Google 試算表（Apps Script 後端），照片存於 Google 雲端硬碟。

## 功能特色

- **直排由右而左**：國語課本式直排，中文字自動標注音（右側、比例縮小），破音字與台灣教育部讀音已處理（如「期」ㄑㄧˊ），可逐字手動修正。
- **課本式排版細節**：括號自動轉直排上下括號（︵︶）；頁碼等英數以「縱中橫」壓成一個中文字寬（如 P.160 ︱ P.161 上下堆疊）；英文與符號不注音。
- **黑板投影**：全螢幕自動適配不溢出；視窗模式可 A−/A＋ 縮放；日期以中文數字呈現（六月十五日）。
- **每日成語**：文字＋解釋＋照片，解釋過長自動折欄。
- **常用連結 App 圖示牆**：可上傳自訂圖示、觸控拖曳排序，手機/平板/桌機響應式欄數。
- **數位作業繳交**：老師勾選「數位作業」的項目才開放線上繳交；家長手機填姓名、勾項目、可附照片。
- **繳交登記表**：學藝股長點選稽核方塊確認繳交，統計各項人數與未交齊名單；可標記「免繳」（特殊生作業調整）；匯出 CSV。
- **未補繳儀表板**：跨日期列出每位學生未繳作業與日期，補繳勾稽即寫回當日紀錄；亦可逐筆免繳。
- **家長專用連結**：網址加 `#parent` 即為唯讀家長頁（僅聯絡簿、連結、繳交），看不到編輯與登記。
- **離線函式庫**：注音與拖曳函式庫隨站台提供，不依賴外部 CDN，校園網路封鎖 CDN 也不受影響。

## 專案結構

```
├── index.html          # 前端主程式（Netlify 發布）
├── pinyin-pro.js       # 注音函式庫（離線版）
├── sortable.min.js     # 拖曳排序函式庫（離線版，僅老師端載入）
└── backend/
    └── Code.gs         # Google Apps Script 後端（貼到試算表的 Apps Script，不會被 Netlify 使用）
```

## 部署步驟

### 一、建立 Google 試算表後端

1. Google 雲端硬碟新增空白試算表（如「302 聯絡簿資料」）。
2. 擴充功能 ▸ Apps Script，貼入 `backend/Code.gs`，存檔。
3. 部署 ▸ 新增部署作業 ▸ 類型「網頁應用程式」：執行身分＝**我**、存取權＝**任何人** ▸ 部署並授權。
4. 複製產生的網頁應用程式網址（結尾 `/exec`）。

工作表（功課／成語／連結／繳交／叮嚀／名單／稽核）會自動建立。**班級座號名單**請直接在「名單」工作表的「姓名」欄逐列填入。

### 二、填入 API 網址

編輯 `index.html` 最上方：

```js
const API_URL = "https://script.google.com/macros/s/XXXX/exec";
```

留空則僅使用瀏覽器本機暫存（測試用）。

### 三、部署到 Netlify（由 GitHub 自動部署）

1. 將本倉庫推上 GitHub。
2. Netlify ▸ Add new site ▸ **Import an existing project** ▸ 選擇此 GitHub 倉庫。
3. Build 設定全部留空（純靜態）：Build command 空白、Publish directory 填 `.`（根目錄）。
4. Deploy。之後每次 `git push`，Netlify 會自動重新部署。

> 也可不串 GitHub，直接把 `index.html`、`pinyin-pro.js`、`sortable.min.js` 三檔拖放到 Netlify 手動部署。

### 四、分享給家長

老師編輯頁「📋 複製家長連結」會複製 `https://你的站名.netlify.app/#parent`，貼到班級群組即可。

## 更新後端注意

修改 `Code.gs` 後，Apps Script 需「管理部署作業 ▸ 編輯 ▸ 版本選新版本 ▸ 部署」才會生效（網址不變）。

## 常見問題

**注音有個別字錯？** 老師編輯清單點該行「注音 ✎」手動修改；也可在 `index.html` 上方 `TW_DICT` 增補台灣讀音詞條。

**家長看不到圖片？** 圖片以雲端硬碟縮圖網址提供（免登入可看）。若仍破圖，確認 Apps Script 已授權雲端硬碟並重新部署新版本。

**數位作業繳交區沒出現？** 只有當天存在被勾選「數位作業」的功課時，繳交區才會顯示。

**特殊生作業調整？** 在「未補繳」按該筆「免繳」，或於登記表點「免」徽章取消；免繳不列入未交也不算已交。

**校園網路很嚴格？** 本站不依賴外部 CDN；僅需能連 Netlify（站台本身）、Google Apps Script 與雲端硬碟（資料與圖片）。

## 開發注意事項（血淚教訓，接手必讀）

### 日期格式防禦守則

本專案曾因日期格式踩過大坑：試算表儲存格「顯示」`2026-07-11`，實際儲存的卻是文字 `Sat Jul 11 2026 00:00:00 GMT+0800 (Taiwan Standard Time)`，導致比對永遠失敗、家長端讀不到資料。防禦原則：

1. **儲存**：日期一律存 `yyyy-MM-dd` 純文字。所有寫入在 `doPost` 進入點以 `normDate()` 正規化。
2. **讀取比對**：`dateMatch()` 寬容比對三種形態——乾淨字串、Date 物件、可解析的日期長字串——統一格式化後比對，並同時嘗試台北／試算表／GMT 時區。
3. **不信任眼睛**：試算表「顯示值」≠「儲存值」。除錯一律印出原始值與型別（`instanceof Date`）。
4. **時區明確化**：`Utilities.formatDate` 一律指定 `Asia/Taipei`，不依賴試算表預設時區。

### Apps Script 部署紀律

- 更新程式 = 「部署 ▸ 管理部署作業 ▸ 鉛筆 ✏️ ▸ 版本選**建立新版本** ▸ 部署」，網址不變。
- **絕不**按「新增部署作業」——會產生新網址，前端 `API_URL` 立刻對不上。
- 保持**一個進行中部署**，其餘封存；封存區的網址視為失效，不要複製使用。
- 編輯器存檔 ≠ 生效，一定要部署新版本才會上線。

### 快速診斷

- **API 是否正常**：瀏覽器開 `API_URL?action=load&date=今天日期&callback=t`，看回應是否含當日功課。回應開頭必須是 `t(`；若是 `callback(` 代表參數沒進去（網址斷了）。
- **一鍵健檢**：Apps Script 編輯器選 `selfTest` ▸ 執行 ▸ 看執行紀錄。會自動檢查試算表綁定、工作表表頭、日期三形態比對、逐列型別、doGet 模擬回應——任一 ✗ 即為病灶。
- **前端指向哪個後端**：正式網頁按 Ctrl+U ▸ 搜 `API_URL`，比對部署 ID 是否為進行中的那一個。

## 隱私提醒

家長連結為「知道網址即可開啟」，無登入驗證。請勿在聯絡簿填寫不宜公開的個資；學生繳交僅記錄姓名、完成狀態與自願上傳的作業照片。
